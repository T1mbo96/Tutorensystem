Eine Funktion ist ein Konstrukt, welches es erlaubt, den Programmcode zu strukturieren und wiederverwendbar zu machen. Ohne die Nutzung von Funktionen müsste man den Code immer wieder kopieren und einfügen sowie bei Änderungen jedes mal erneut anpassen. Eine Funktion kapselt also einen Task, den man immer wieder braucht und macht ihn, ohne den Code erneut schreiben zu müssen, nutzbar.

Zusammengesetzt ist eine Funktion aus dem Rückgabewert, dem Funktionsnamen und den Parametern, welche die Funktion erhält, um damit zu arbeiten. Allerdings muss eine Funktion nicht zwangsweise einen Rückgabewert oder Parameter besitzen, wie sie in den kommenden Beispielen sehen werden.

Funktionen mit Rückgabewert

Eine Funktion mit Rückgabewert besitzt, wie der Name schon sagt, einen Wert, den Sie, nachdem die Anweisungen innerhalb der Funktion abgearbeitet sind, zurückgibt. Diesen Wert kann man direkt benutzen, sodass Funktionen auch in Ausdrücken verwendet werden können. In Blockly wird eine Funktion folgendermaßen erzeugt.

Sie können beliebig viele Inputs deklarieren, mit denen die Funktion anschließend arbeiten kann. Diese Inputs sind die Parameter einer Funktion, welche beim Aufruf einer Funktion auch angegeben werden müssen. Der Abschnitt do something ist der Funktionsname und sollte so gewählt werden, dass er die Aufgabe der Funktion beschreibt. Das Kästchen allow statements lässt Sie auswählen, ob Sie innerhalb der Funktion Anweisungen zulassen wollen oder nicht. Dies ist hilfreich, wenn Sie nur eine einzelne Anweisung brauchen, die Sie direkt an das return hängen. Das return gibt dann einen Wert, den Sie bestimmen können, zurück.
