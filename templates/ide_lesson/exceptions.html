Ausnahmen

Eine Ausnahme (engl. Exception) ist, wie der Name schon vermuten lässt, ein Ereignis, welches den normalen Ablauf eines Programms während der Ausführung stört. Dies bedeutet, wenn ein Python-Programm auf eine Situation stößt, mit der es nicht umgehen kann, dann wird eine Ausnahme ausgelöst (z.B. das Programm versucht auf einen Index einer Liste zuzugreifen, welcher überhaupt nicht existiert). Eine Ausnahme ist ein Python-Objekt, welches einen Fehler abbildet.

Falls eine Ausnahme in der Ausführung eines Programms nicht behandelt wird, wird das Programm beendet. Um eine Ausnahme zu behandeln, wird der Code, welcher eine Ausnahme hervorrufen könnte, in einem sogenannten try-Block platziert, gefolgt von einem except-Statement. Das except-Statement soll hierbei den Fehler so spezifisch wie möglich abfangen. Es wird generell als Bad Practice angesehen einfach except Exception: oder nur except: zu verwenden.

Um diesen Fehler zu behandeln, sodass unser Programm nicht jedes Mal abbricht, wenn versucht wird auf einen Index zuzugreifen, der nicht existiert, platzieren wir den Code in einem try-Block.

Diesmal wird die Ausnahme behandelt und Code raised error! in der Konsole ausgegeben, ohne dass das Programm beendet wird.

Multiple except-Statements

Sie können auch mehrere Ausnahmen abfangen und diese unterschiedlich behandeln.

else-Block

Außerdem gibt es einen else-Block für den try-Block. Dieser wird ausgeführt, falls keine Ausnahme aufgetreten ist.

Ausnahme werfen

Sie können auch selbst bestimmen, wann eine spezifische Exception in ihrem Code geworfen werden soll. Dafür verwenden Sie an entsprechender Stelle das Schlüsselwort raise, gefolgt von der Bezeichnung der Ausnahme, die geworfen werden soll.
